SoulDrive 项目架构图

SoulDrive 是一个本地优先的桌面知识引擎：用可移动存储作为授权和数据入口，后端负责硬件监听、文档入库、向量与图谱检索、本地 LLM 推理，前端通过 Tauri + React 提供聊天、SSE 流式输出和结构化导图工作台。

前端/桌面壳 运行态与鉴权 入库管线 Hybrid RAG 推理 本地数据/模型

一、总体架构

按“设备事件 -> 知识入库 -> 检索推理 -> 前端展示”的闭环组织。

二、模块/功能拆分

每个模块对应当前代码中的主要文件和职责边界。

core/hardware_watcher.py

硬件事件监听

  • 用 psutil.disk_partitions() 轮询可移动盘。
  • 插入后触发鉴权、运行态解锁、后台索引。
  • 拔出后锁定运行态，通知 API 清理资源。
core/hardware_security.py

硬件授权

  • 通过 PowerShell WMI 读取 USB 设备信息。
  • 按 VID/PID、型号、序列号判断 PRO/LITE。
  • 为运行态提供硬件 SN 和授权级别。
core/indexer.py

增量入库调度

  • 递归扫描 U 盘 PDF。
  • 用 MD5 判断是否已入库。
  • 组织解析、向量写入、PRO 图谱抽取。
core/document_parser.py

文档解析切片

  • Docling 将 PDF 转为 Markdown。
  • 按 Markdown 标题进行语义切分。
  • 保留更适合 RAG 的段落结构。
core/knowledge_base.py

向量知识库

  • 加载本地 BGE-Small 中文 embedding。
  • 持久化写入 ChromaDB。
  • 检索时召回文本块并补充图谱上下文。
core/graph_db.py

本地图谱库

  • SQLite 模拟轻量图数据库。
  • entities / relationships 两张核心表。
  • 支持实体匹配和一跳/多跳关系查询。
core/graph_extractor.py

图谱抽取

  • 本地 GGUF 模型从文本块抽实体和关系。
  • 要求 JSON 结构化输出并做容错修复。
  • 仅 PRO 模式在入库阶段启用。
core/rag_engine.py

Hybrid RAG 推理

  • 合并向量片段和图谱关系链。
  • 用本地 llama.cpp 加载 Qwen GGUF。
  • 通过生成器向 FastAPI 流式返回答案。
core/mcp_server.py

FastAPI 服务

  • /chat 接收问题并返回 SSE。
  • /runtime/* 提供锁定、解锁、状态查询。
  • 懒加载知识库和推理引擎。
core/runtime_state.py

运行态闸门

  • 用 souldrive_db/runtime_state.json 持久化。
  • 记录 locked、auth_level、active_drive。
  • 拔盘后阻断聊天并触发资源清场。
souldrive-ui/src/App.tsx

React 前端

  • 左侧聊天输入和 Markdown 渲染。
  • 轮询运行态，锁定时禁用输入。
  • 解析 souldrive-mindmap 代码块并绘图。
souldrive-ui/src-tauri

Tauri 桌面壳

  • 提供跨平台桌面窗口。
  • 当前 Rust 侧以基础 shell 和 opener 插件为主。
  • 主要业务仍在 Python 服务和 React 内完成。

四、设计思路

这部分解释为什么项目按当前方式拆分。

1. 本地优先

模型、向量库、图谱库和运行态都落在本机目录。优点是隐私边界清晰，离线可用；代价是初始化模型和解析 PDF 会占用本地 CPU/内存。

2. 硬件即权限闸门

U 盘既是数据来源，也是授权事件来源。插入时解锁并入库，拔出时锁定服务、清理模型和数据库连接，形成物理层面的安全闭环。

3. 双轨知识结构

ChromaDB 负责“找到相关原文细节”，SQLite 图谱负责“补充实体关系链”。RAGEngine 把两类上下文合并后交给本地 LLM。

4. 前后端解耦

Python 后端通过 HTTP/SSE 暴露能力，React 只关心输入、流式展示和导图渲染，Tauri 负责桌面容器。这让 UI 和推理服务可以分别演进。

五、当前实现边界

这些点有助于阅读代码或继续演进项目。

已落地的主链路

  • 可移动盘监听、PRO/LITE 鉴权、运行态锁定/解锁。
  • PDF 增量入库、Docling 解析、ChromaDB 持久化。
  • PRO 模式图谱抽取，SQLite 存实体和关系。
  • FastAPI SSE 聊天、本地 GGUF 推理、React 导图工作台。

需要注意的边界

  • 硬件指纹读取依赖 Windows PowerShell WMI。
  • 当前 Tauri Rust 侧业务很轻，主要能力在 Python/React。
  • 图谱抽取和问答共用本地 GGUF，资源管理是核心稳定性问题。
  • 部分源码注释存在编码显示异常，逻辑仍可按代码结构阅读。
